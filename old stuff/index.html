FEATURES

In pursuit of this critical task, we carefully selected specific sensors to equip our CLAWNTLET, each chosen for its unique capabilities and contributions to the success of our mission.

  • State of the Art Sensors
    •

    As we navigate through Boelter Hall, several state of the art sensors will gather and display information on an OLED screen placed on the CLAWNTLET. It includes a Gas Detection Module, Ultrasonic Sensor, NeoPixel Ring, and Temperature/Humidity Sensor.

  • Gas Detection Module: able to read the potentially harmful gasses in a room and relays the information to the CLAWNTLET for the user to see.
  • Ultrasonic Sensor: used to calculate the distance in cm between an object and the CLAWNTLET to be displayed on the OLED screen.
  • NeoPixel Ring: used to warn the user of potentially harmful conditions by lighting up 12 LEDs with specific colors to indicate danger levels.
  • Temperature/Humidity Sensor: able to read the temperature and humidity in a room and relays the information to the CLAWNTLET for the user to see.

  • OLED Screen

    • The CLAWNTLET features a state of the art OLED screen that reads the various sensors located on the CLAWNTLET and displays the information for the user to see.

    CODE

    Full code can be found on our GitHub.

    We designed our code with the victims in Boelter Hall in mind. Using the various electronics connected to the ESP32, the code is responsible for communicating with the sensors, OLED screen, and even this website. The code is uploaded to the ESP32 using the Arduino IDE and is the heart of our CLAWNTLET.

    • ESP32 with Arduino IDE

      • Code is uploaded onto the ESP32 using the Arduino IDE and is powered by a mobile power bank.

      As with all coding, it took a lot of debugging to get it working. An example of an issue we encountered with the ESP32 was that it had only one SDA and one SCL pin. This led to errors since both the OLED and temperature/humidity sensor required these pins. However, after conducting further research, we discovered that it is possible to share the SDA and SCL pins, which resolved the problem.
